16     package com.google.errorprone.annotations;

18     import static java.lang.annotation.ElementType.TYPE;
19     import static java.lang.annotation.RetentionPolicy.RUNTIME;
20     
21     import java.lang.annotation.Retention;
22     import java.lang.annotation.Target;
23     
24     /**
25      * The class to which this annotation is applied is immutable.
26      *
27      * <p>An object is immutable if its state cannot be observed to change after construction. Immutable
28      * objects are inherently thread-safe.
29      *
30      * <p>A class is immutable if all instances of that class are immutable. The immutability of a class
31      * can only be fully guaranteed if the class is final, otherwise one must ensure all subclasses are
32      * also immutable.
33      *
34      * <p>A conservative definition of object immutability is:
35      *
36      * <ul>
37      * <li>All fields are final;
38      * <li>All reference fields are of immutable type, or null;
39      * <li>It is <em>properly constructed</em> (the {@code this} reference does not escape the
40      * constructor).
41      * </ul>
42      *
43      * <p>The requirement that all reference fields be immutable ensures <em>deep</em>
44      * immutability, meaning all contained state is also immutable. A weaker property, common with
45      * container classes, is <em>shallow</em> immutability, which allows some of the object's fields to
46      * point to mutable objects. One example of shallow immutability is guava's ImmutableList, which
47      * may contain mutable elements.
48      *
49      * <p>It is possible to implement immutable classes with some internal mutable state, as long as
50      * callers can never observe changes to that state. For example, some state may be lazily
51      * initialized to improve performance.
52      *
53      * <p>It is also technically possible to have an immutable object with non-final fields (see the
54      * implementation of {@link String#hashCode()} for an example), but doing this correctly requires
55      * subtle reasoning about safe data races and deep knowledge of the Java Memory Model.
56      *
57      * Use of this annotation is validated by
58      * <a href="http://errorprone.info/bugpatterns/ImmutableChecker">Error Prone's immutability analysis</a>,
59      * which ensures that all {@code @Immutable}-annotated classes are deeply immutable according to the
60      * conservative definition above. Non-final classes may be annotated with {@code @Immutable}, and
61      * any code compiled by Error Prone will be checked to ensure that no mutable subtypes of
62      * {@code @Immutable}-annotated classes exist.
63      *
64      * <p>For more information about immutability, see:
65      *
66      * <ul>
67      * <li>Java Concurrency in Practice ยง3.4
68      * <li>Effective Java ยง15
69      * </ul>
70      */
71     @Target(TYPE)
72     @Retention(RUNTIME)
73     public @interface Immutable {
74     
75       /**
76        * When annotating a generic type as immutable, {@code containerOf} specifies
77        * which type parameters must be instantiated with immutable types for the
78        * container to be deeply immutable.
79        */
80       String[] containerOf() default {};
81     }
