145     /* An array of strings harvested from --require-text-symbol= 
146        flags.
147     
148        Each string specifies a pair: a soname pattern and a text symbol
149        name pattern, separated by a colon.  The patterns can be written
150        using the normal "?" and "*" wildcards.  For example:
151        ":*libc.so*:foo?bar".
152     
153        These flags take effect when reading debuginfo from objects.  If an
154        object is loaded and the object's soname matches the soname
155        component of one of the specified pairs, then Valgrind will examine
156        all the text symbol names in the object.  If none of them match the
157        symbol name component of that same specification, then the run is
158        aborted, with an error message.
159     
160        The purpose of this is to support reliable usage of marked-up
161        libraries.  For example, suppose we have a version of GCC's
162        libgomp.so which has been marked up with annotations to support
163        Helgrind.  It is only too easy and confusing to load the 'wrong'
164        libgomp.so into the application.  So the idea is: add a text symbol
165        in the marked-up library (eg), "annotated_for_helgrind_3_6", and
166        then give the flag
167     
168          --require-text-symbol=:*libgomp*so*:annotated_for_helgrind_3_6
169     
170        so that when libgomp.so is loaded, we scan the symbol table, and if
171        the symbol isn't present the run is aborted, rather than continuing
172        silently with the un-marked-up library.  Note that you should put
173        the entire flag in quotes to stop shells messing up the * and ?
174        wildcards. */
175     extern Int    VG_(clo_n_req_tsyms);
176     extern HChar* VG_(clo_req_tsyms)[VG_CLO_MAX_REQ_TSYMS];
